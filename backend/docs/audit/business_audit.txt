================================================================================
BUGS FOUND AND FIXED
================================================================================

BUG 1: Criteria duplicate data → unhandled 500
  File: app/api/v1/critaria/service.py
  Root cause: criteria.data column has UNIQUE constraint. Creating the same
    criteria data twice raised psycopg2.errors.UniqueViolation which was
    not caught, resulting in a 500 Internal Server Error.
  Fix: Wrapped create() in try/except IntegrityError → raises 409 Conflict.
    Added db.rollback() to prevent session poisoning.

BUG 2: AI assistant 502 when Together AI model unavailable
  File: app/api/v1/assistant/service.py
  Root cause: The fallback to Groq only triggered on specific error strings
    ("dedicated_endpoint_not_running", "not running", "model_not_available").
    The actual Together error was "Unable to access non-serverless model"
    (BadRequestError 400), which did not match any keyword, so Groq fallback
    was never attempted and the API returned 502.
  Fix: Changed to always fall back to Groq on any Together exception.
    Removed the narrow keyword-matching gate.

  Extension: .jpg, .jpeg, .png, .webp only
  Content-type: image/jpeg, image/png, image/webp only (ADDED in this audit)
  Previously only extension was checked — a malicious file named .jpg
  with a non-image content-type would pass through.

[PASS] API key revocation is immediate and enforced
  Revoked keys return 401 on next use.

[PASS] JWT token scoping — no cross-company data leakage
  Company 2's API key only returns Company 2's data.
